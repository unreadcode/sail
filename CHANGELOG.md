# 更新日志

> 发版流程：在此文件顶部新增一节 `## <版本号>`（与即将打的 `v<版本号>` tag 对应），
> 写本次更新内容；CI（`.github/workflows/release.yml`）按 tag 自动提取对应小节注入 GitHub Release，
> App「设置 › 关于」在检测到新版本时展示这段内容。版本号与 `MARKETING_VERSION` 保持一致。

## 1.1.19

- 修复关闭 TUN 后内核仍反复重启、日志刷 `address already in use` 的问题：切换时未等旧内核真正释放端口就启动新内核，现改为确认混合端口空出后再启动，无论旧内核退出快慢都不会再撞端口。
- 加固 TUN 特权组件：停止内核时按可执行路径清除可能遗留的内核进程（覆盖特权组件重装 / 重启遗留的残留）。
- 注：本版升级了 TUN 特权组件，已启用 TUN 的用户更新后首次启动会弹一次管理员授权以同步组件，属正常现象。

## 1.1.18

- 修复关闭 TUN（虚拟网卡）模式时内核反复重启、日志刷 `address already in use`（并发启动把同一端口双开占用）。
- 修复导入个别格式异常的订阅（含超大端口字段）时 App 崩溃。
- 修复检查更新 / 关闭更新窗口后 App 偶发崩溃。
- 修复节点数量很多的大型订阅在 TUN 模式下可能启动失败。
- 修复下载更新时关闭主窗口会连带隐藏更新窗口、以及关闭更新窗口影响主窗口。
- 加固 TUN 特权 helper：收紧文件描述符继承、缩小内核启动竞态窗口。
- 系统代理设置失败时记录诊断日志，不再静默吞掉。
- 注：本版升级了 TUN 特权组件，已启用 TUN 的用户更新后首次启动会弹一次管理员授权以同步组件，属正常现象。

## 1.1.17

- 修复内核偶发异常退出（code 1）：关闭代理 / 切换开关时，clash_api 端口与上一内核实例残留的 TIME_WAIT 状态冲突（`address already in use`）。现改为每次启动选用新的空闲端口，并避开其它 Clash 客户端固定占用的 9090。

## 1.1.16

- 内核改为随版本内置分发，移除 App 内联网下载/更新入口（发新版即更新内核）。
- 修复 TUN 模式下每次启动都弹管理员密码：helper 版本检查改为自愈，杜绝版本号两处漂移。
- 设置页移除「内核」分区，内核版本移至「关于」页展示。
- 内置 sing-box 升级至 1.13.14。
- 修复 macOS 15 上添加 / 编辑订阅时，长订阅链接在输入框被截断。
- 日志页：手动上滚查看历史时不再被持续输出强制拽回底部，并提供「回到底部」按钮。
- 检测到新版本时，侧栏图标显示 NEW 角标，点击打开更新窗口。
- 更新改为独立窗口：展示本版更新日志 + 取消 / 更新，点「更新」在窗口底部就地显示下载 / 安装进度（「设置 › 关于」的「查看更新」同此窗口）。
